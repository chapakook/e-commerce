# 🛒 E-Commerce Ubiquitous Language
## 👤 Users (유저)
- `User`: 이커머스를 이용하는 개인, 고유한 `ID`와 `이메일`로 식별됨.
- `UserId`: 유저를 구분하는 고유 키.
- `Profile`: 유저가 조회할 수 있는 개인 정보(`성별`, `생년월일`, `이메일`).

## 💰 Points (포인트)
- `Point`: 유저가 보유한 가상화폐, 현금처럼 사용 가능.
- `Balance`: 현재 보유 포인트 잔액.
- `History`: 포인트 충전, 사용, 환불 등의 내역 기록.
- `Deposit` (충전): 포인트를 계정에 추가하는 행위.
- `Capture`: 주문 시 포인트 사용 확정.
- `Release`: 결제 실패/주문 취소 시 포인트 복구.

## 🏷️ Brands (브랜드)
- `Brand`: 특정 상품을 판매하는 회사/제조사/상표.
- `BrandId`: 브랜드 식별자.
- `BrandName`: 브랜드 고유명 (전역 유일).

## 🛍️ Products (상품)
- `Product`: 브랜드에 속하는 기본 단위 상품.
- `Option (옵션)`: 색상/사이즈 등 구체적인 선택 요소.
- `SKU (옵션상품)`: 옵션 조합으로 정의된 단위.
- `UNI-SKU (마스터상품)`: 고객이 보는 대표 상품 단위.
- `Price`: 상품 가격, SKU 기준으로 결정됨.
- `Stock (재고)`: SKU 단위 수량.

## ❤️ Likes (좋아요)
- `Like`: 유저가 특정 상품에 표현하는 선호 표시.
- `ProductLike`: 유저-상품 간의 좋아요 관계.
- `TotalLikes`: 특정 상품의 누적 좋아요 수.

## 🧾 Orders/Payments (주문/결재)
- `Order`: 유저가 상품을 구매하기 위해 생성하는 거래 단위.
- `OrderItem`: 주문에 포함된 SKU와 수량.
- `Pre-Order (가주문)`: 결제가 완료되기 전 임시 주문 상태.
- `Confirmed Order (확정 주문)`: 결제 성공 시 확정된 주문.
- `Canceled Order`: 실패/취소로 무효화된 주문.
- `Payment`: 결제 프로세스 전체(포인트, PG 결제 포함).
- `PaymentAttempt`: 개별 결제 시도(성공/실패).

## 🛒 Cart (장바구니)
- `Cart`: 유저별로 관리되는 상품 모음.
- `CartItem`: 장바구니에 담긴 SKU와 수량.
- `CartCheckout`: 장바구니 기반으로 주문을 생성하는 행위.

## 🎟️ Coupons (쿠폰)
- `Coupon`: 주문 시 적용할 수 있는 할인 수단.
- `CouponType`: 할인 방식 (정액, 정률).
- `UserCoupon`: 특정 유저가 발급받은 쿠폰.
- `CouponHold`: 주문 도중 쿠폰을 임시로 묶어두는 상태.
- `CouponConsume`: 주문 확정 시 쿠폰 사용 처리.
- `CouponRestore`: 주문 취소 시 쿠폰을 다시 유효 상태로 되돌림.

## 🔗 공통 용어
- `Discount`: 쿠폰이나 프로모션을 통한 가격 차감.
- `Reservation` (예약): 재고나 포인트, 쿠폰을 잠정적으로 확보해두는 상태.
- `Commit`: 예약된 자원을 확정.
- `Release`: 예약된 자원을 해제.
- `Idempotency` (멱등성): 같은 요청을 여러 번 해도 결과가 변하지 않는 성질.
- `Aggregate`: 비즈니스 규칙을 담는 응집된 도메인 객체 집합 (예: Order, Product, User).
- `Event`: 상태 변경을 알리는 도메인 신호 (예: OrderPlaced, PointsCaptured).
